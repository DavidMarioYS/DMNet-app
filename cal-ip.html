Fundamental IP Address

IP (Internet Protocol) Address adalah alamat logika yang diberikan kepada perangkat agar bisa berkomunikasi di jaringan. Berbeda dengan MAC Address yang merupakan "sidik jari" fisik dari pabrik, IP Address bersifat dinamis.

Public vs Private IP

  • 🌍
    IP Public: Bisa diakses langsung dari internet. Disediakan ISP.
  • 🏠
    IP Private: IP lokal (LAN). Butuh NAT untuk ke internet.

🔍 Anatomi Subnetting

Network ID

Ibarat "Nama Jalan". Ini adalah alamat pertama dalam subnet yang berfungsi sebagai identitas jaringan. Tidak bisa dipasang ke komputer.

Host ID (Usable)

Ibarat "Nomor Rumah". Rentang alamat IP valid di tengah yang bisa dipasangkan ke PC, Laptop, atau Router.

Broadcast Address

Ibarat "Toa Pengumuman". Ini adalah alamat terakhir dalam subnet. Paket data ke IP ini akan diterima oleh semua komputer di jaringan tersebut.

⚠️ IP Khusus (Reserved IP)

Loopback

127.0.0.1

Untuk menguji network adapter di komputer sendiri (Localhost).

APIPA

169.254.x.x

Dibuat otomatis oleh Windows jika gagal mendapat IP dari DHCP Server.

Default Route

0.0.0.0/0

Mewakili "semua rute di internet". Sering dipakai di routing tabel.

IPv6 Localhost

::1

Sama seperti 127.0.0.1 namun ini adalah versi IPv6.

🚀 Masa Depan: Transisi ke IPv6

Karena alamat IPv4 sudah habis, dunia beralih ke IPv6. Formatnya bukan lagi desimal 32-bit, melainkan Heksadesimal 128-bit. Kapasitasnya hampir tak terbatas!

Format Dasar IPv6

Format Kompresi (Disingkat)

🧮 Contoh Perhitungan Subnetting

Pelajari cara memecah jaringan secara manual, langkah demi langkah.

Contoh 1

IP: 192.168.10.0 / 24 — Berapa host yang bisa dipakai?

Langkah Perhitungan

1
Prefix /24 → host bits = 32 - 24 = 8 bit
2
Total IP = 2⁸ = 256 alamat
3
Host usable = 256 - 2 = 254 (dikurangi Network ID & Broadcast)
4
Subnet mask = 255.255.255.0
Network ID192.168.10.0
IP Pertama192.168.10.1
IP Terakhir192.168.10.254
Broadcast192.168.10.255
Host Usable254 perangkat
Contoh 2

IP: 192.168.1.0 / 26 — Berapa subnet & host tiap subnet?

Langkah Perhitungan

1
Host bits = 32 - 26 = 6 bit
2
Tiap subnet = 2⁶ = 64 IP, usable = 62 host
3
Jumlah subnet = 256 ÷ 64 = 4 subnet
4
Block size / interval = 64

4 Subnet yang Terbentuk

Subnet 1: 192.168.1.0 — 192.168.1.63
Net: .0 | Usable: .1-.62 | BC: .63
Subnet 2: 192.168.1.64 — 192.168.1.127
Net: .64 | Usable: .65-.126 | BC: .127
Subnet 3: 192.168.1.128 — 192.168.1.191
Net: .128 | Usable: .129-.190 | BC: .191
Subnet 4: 192.168.1.192 — 192.168.1.255
Net: .192 | Usable: .193-.254 | BC: .255
Contoh 3

VLSM: 172.16.0.0/16 untuk 3 divisi (100, 50, 20 host)

VLSM (Variable Length Subnet Mask) menggunakan prefix berbeda tiap divisi agar tidak boros IP. Urutkan dari kebutuhan terbesar ke terkecil!

DivisiKebutuhanPrefixNetwork IDRange UsableHost Tersedia
Produksi100 host/25172.16.0.0172.16.0.1 – .126126
Pemasaran50 host/26172.16.0.128172.16.0.129 – .19062
Keuangan20 host/27172.16.0.192172.16.0.193 – .22230
💡 Tips VLSM: Rumus mencari prefix → cari 2ⁿ ≥ (kebutuhan host + 2). Misal 100 host → 2⁷=128 → prefix /25.

🔗 Protokol Jaringan Penting

Protokol-protokol yang bekerja di balik layar setiap jaringan komputer.

🌐

DNS

Port 53

Menerjemahkan nama domain (google.com) ke IP address (142.250.x.x). Ibarat buku telepon internet.

google.com → 142.250.68.46
📦

DHCP

Port 67/68

Memberikan IP address secara otomatis ke perangkat yang bergabung ke jaringan. Tanpa DHCP, semua IP harus diatur manual.

DORA: Discover → Offer → Request → Ack
🔁

NAT

Layer 3

Menerjemahkan IP Private ke IP Public saat mengakses internet. Router di rumah selalu menggunakan NAT.

192.168.1.5 → 103.24.x.x (ISP)
📡

ARP

Layer 2

Address Resolution Protocol — menemukan MAC Address dari sebuah IP Address di jaringan lokal (LAN). Bekerja di Layer 2.

"Siapa yang punya 192.168.1.1?" → MAC: AA:BB:CC:...
🏓

ICMP

Layer 3

Internet Control Message Protocol — dipakai oleh perintah ping dan traceroute untuk menguji konektivitas dan mengukur latensi antar host.

ping 8.8.8.8 → Reply from 8.8.8.8: bytes=32 time=12ms
🔀

Routing

Layer 3

Proses menentukan jalur terbaik paket data dari sumber ke tujuan antar jaringan berbeda menggunakan protokol seperti OSPF, RIP, atau BGP.

10.0.1.0/24 via 10.0.0.1 (gateway)

🏛️ Classful (Lama)

Sebelum 1993 — kaku, boros IP

✗Hanya ada Class A, B, C — tidak bisa dikustom
✗Class B (65K host) diberikan ke perusahaan yang hanya butuh 1000 host → 64K IP terbuang
✗Tidak mendukung subnet dengan ukuran fleksibel

Contoh boros: butuh 300 host → ambil Class B (65K) → waste 65K IP!

⚡ CIDR (Modern)

Sejak 1993 — fleksibel, efisien

✓Prefix bebas dari /1 sampai /32
✓Butuh 300 host? Pakai /23 (510 host) — jauh lebih efisien
✓Mendukung supernetting & route aggregation di internet

Contoh efisien: butuh 300 host → pakai /23 (510 host) → hemat!

💡 Tips & Trik Subnetting

Rumus dan shortcut yang wajib dihafal untuk ujian dan dunia kerja.

Rumus Host

2ⁿ - 2

n = jumlah bit host. Dikurangi 2 untuk Network ID dan Broadcast.

Rumus Subnet

2ˢ

s = jumlah bit yang "dipinjam" dari host bits untuk subnet.

Block Size

256 - mask

Contoh: mask oktet = 192 → block = 256-192 = 64. Interval antar subnet.

🎯 Cara Cepat Menentukan Prefix dari Kebutuhan Host: Cari nilai 2ⁿ yang paling kecil tapi ≥ (kebutuhan + 2). Contoh: butuh 60 host → 2⁶=64 ≥ 62 ✓ → prefix = 32-6 = /26.

IP Address Fundamentals

An IP Address is a logical address assigned to devices to communicate. Unlike a MAC Address which is a physical hardware "fingerprint", an IP Address is dynamic.

Public vs Private IP

  • 🌍
    Public IP: Recognized globally. Provided by ISP.
  • 🏠
    Private IP: Internal networks (LAN). Requires NAT.

🔍 Subnet Anatomy

Network ID

Like a "Street Name". It's the first address in a subnet representing the network identity. Cannot be assigned to a PC.

Host ID (Usable)

Like a "House Number". The valid range of IPs in the middle that can be assigned to devices.

Broadcast Address

Like a "Megaphone". The last address. Packets sent here go to all computers in the subnet.

⚠️ Reserved IPs

Loopback

127.0.0.1

Used to test the local machine's network adapter (Localhost).

APIPA

169.254.x.x

Auto-assigned by Windows if DHCP fails.

Default Route

0.0.0.0/0

Represents "all routes". Used heavily in routing tables.

IPv6 Localhost

::1

The IPv6 equivalent of 127.0.0.1.

🚀 The Future: IPv6 Transition

Since IPv4 addresses are exhausted, the world is moving to IPv6. It uses a 128-bit Hexadecimal format, providing near-infinite capacity.

Standard Format

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Compressed Format

🧮 Subnetting Calculation Examples

Learn to manually calculate subnets, step by step.

Example 1

IP: 192.168.10.0 / 24 — How many usable hosts?

Calculation Steps

1
Prefix /24 → host bits = 32 - 24 = 8 bits
2
Total IPs = 2⁸ = 256 addresses
3
Usable hosts = 256 - 2 = 254 (minus Network ID & Broadcast)
4
Subnet mask = 255.255.255.0
Network ID192.168.10.0
First Host192.168.10.1
Last Host192.168.10.254
Broadcast192.168.10.255
Usable Hosts254 devices
Example 2

IP: 192.168.1.0 / 26 — How many subnets & hosts per subnet?

Calculation Steps

1
Host bits = 32 - 26 = 6 bits
2
Each subnet = 2⁶ = 64 IPs, usable = 62 hosts
3
Number of subnets = 256 ÷ 64 = 4 subnets
4
Block size / interval = 64

4 Subnets Created

Subnet 1: 192.168.1.0 — 192.168.1.63
Net: .0 | Usable: .1-.62 | BC: .63
Subnet 2: 192.168.1.64 — 192.168.1.127
Net: .64 | Usable: .65-.126 | BC: .127
Subnet 3: 192.168.1.128 — 192.168.1.191
Net: .128 | Usable: .129-.190 | BC: .191
Subnet 4: 192.168.1.192 — 192.168.1.255
Net: .192 | Usable: .193-.254 | BC: .255
Example 3

VLSM: 172.16.0.0/16 for 3 departments (100, 50, 20 hosts)

VLSM uses different prefixes per department to minimize IP waste. Always sort from largest to smallest requirement!

DepartmentRequiredPrefixNetwork IDUsable RangeAvailable Hosts
Production100 hosts/25172.16.0.0172.16.0.1 – .126126

🔗 Essential Network Protocols

The protocols working behind the scenes in every computer network.

🌐

DNS

Port 53

Translates domain names (google.com) to IP addresses. Think of it as the internet's phone book.

google.com → 142.250.68.46
📦

DHCP

Port 67/68

Automatically assigns IP addresses to devices joining the network. Without it, every IP must be configured manually.

DORA: Discover → Offer → Request → Ack
🔁

NAT

Layer 3

Translates a Private IP to a Public IP when accessing the internet. Your home router always uses NAT.

192.168.1.5 → 103.24.x.x (ISP)
📡

ARP

Layer 2

Address Resolution Protocol — finds the MAC address of a given IP address on the local LAN. Operates at Layer 2.

"Who has 192.168.1.1?" → MAC: AA:BB:CC:...
🏓

ICMP

Layer 3

Used by ping and traceroute to test connectivity and measure latency between hosts.

ping 8.8.8.8 → Reply: bytes=32 time=12ms
🔀

Routing

Layer 3

Determines the best path for packets across different networks using protocols like OSPF, RIP, or BGP.

10.0.1.0/24 via 10.0.0.1 (gateway)

🏛️ Classful (Legacy)

Pre-1993 — rigid, wasteful

✗Only Class A, B, C — no customization
✗Class B (65K hosts) given to a company needing 1K hosts → 64K IPs wasted
✗No support for variable-size subnets

Wasteful: need 300 hosts → forced to take Class B (65K) → 65K IPs wasted!

⚡ CIDR (Modern)

Since 1993 — flexible, efficient

✓Any prefix from /1 to /32
✓Need 300 hosts? Use /23 (510 hosts) — far more efficient
✓Supports supernetting & route aggregation on the internet

Efficient: need 300 hosts → use /23 (510 hosts) → much less waste!

💡 Subnetting Tips & Tricks

Formulas and shortcuts essential for exams and real-world work.

Host Formula

2ⁿ - 2

n = number of host bits. Subtract 2 for Network ID and Broadcast.

Subnet Formula

2ˢ

s = number of bits "borrowed" from the host portion for subnetting.

Block Size

256 - mask

E.g: mask octet = 192 → block = 256-192 = 64. The interval between subnets.

🎯 Quick Prefix Finder: Find the smallest 2ⁿ that is ≥ (required hosts + 2). Example: need 60 hosts → 2⁶=64 ≥ 62 ✓ → prefix = 32-6 = /26.
